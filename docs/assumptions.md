# Project Assumptions

## Data Assumptions
- Only **successful payments** are considered for active businesses
- Appointment data is assumed to be accurately timestamped
- Training completion status is reliable and up to date

## Business Logic Assumptions
- Engagement is measured using appointment volume
- Month-over-month comparison is sufficient to identify churn risk
- Training impact is evaluated within one month post-completion

## Analytical Assumptions
- External factors (seasonality, holidays) are not modeled
- All businesses are evaluated equally regardless of size
- Missing data is treated as zero activity where applicable

## Limitations
- Predictive modeling is not included in this version
- Insights are based on historical patterns, not causal inference
